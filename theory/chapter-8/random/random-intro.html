    Введение в генерацию случайных чисел

    Случайные числа нужны в играх, симуляциях, криптографии. Но компьютер — детерминированная машина: он не может бросить монету. Вместо этого программы используют псевдослучайные генераторы (PRNG).

    Алгоритмы и состояние

    PRNG — это алгоритм с состоянием: он хранит некоторые данные между вызовами и использует их для генерации следующего числа. Простейший пример:

    C++ — простейший алгоритм с состоянием 
    #include <iostream>

int plusOne()
{
    static int s_state { 3 }; // состояние
    ++s_state;                // изменяем состояние
    return s_state;           // возвращаем следующее число
}

int main()
{
    std::cout << plusOne() << '\n'; // 4
    std::cout << plusOne() << '\n'; // 5
    std::cout << plusOne() << '\n'; // 6
    return 0;
}

    Псевдослучайный генератор

    Настоящий PRNG использует более сложные математические операции, чтобы результат выглядел случайным. Вот упрощённый пример (не используйте в реальном коде):

    C++ — простой PRNG (только для иллюстрации) 
    unsigned int LCG16()
{
    static unsigned int s_state { 0 };
    s_state = 8253729 * s_state + 2396403; // изменяем состояние
    return s_state % 32768;                // возвращаем число в диапазоне [0, 32767]
}

    Инициализация (seed)

    PRNG детерминирован: при одном и том же начальном состоянии он всегда выдаёт одну и ту же последовательность. Начальное состояние задаётся через seed (зерно).

    📐 Как работает seed
    seed = 7  →  PRNG  →  10458, 3853, 16032, 17299, ...
seed = 7  →  PRNG  →  10458, 3853, 16032, 17299, ...  (та же последовательность!)
seed = 9876 → PRNG →  24071, 18138, 27917, ...        (другая последовательность)

    Чтобы программа давала разные числа при каждом запуске, нужно каждый раз использовать разный seed. Обычно для этого берут текущее время или системный генератор случайных чисел.

    Что делает PRNG хорошим

    ✅ Свойства хорошего PRNG
    • Равномерное распределение — каждое число генерируется с одинаковой вероятностью
    • Непредсказуемость — следующее число нельзя угадать по предыдущим
    • Большой период — последовательность повторяется как можно реже
    • Эффективность — быстрая генерация, небольшое состояние

    PRNG в стандартной библиотеке C++

    Заголовок <random> предоставляет несколько PRNG-алгоритмов:

    Тип Семейство Качество Использовать?
    mt19937 Mersenne Twister Хорошее ✅ Да
    minstd_rand Linear congruential Плохое ❌ Нет
    default_random_engine Зависит от реализации Неизвестно ❌ Нет
    rand() Устаревший (из C) Ужасное ❌ Никогда
    Лучшая практика

    Используйте std::mt19937 из заголовка <random>. Никогда не используйте rand().
