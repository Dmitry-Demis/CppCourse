    Header guards

    Если один заголовочный файл включается несколько раз (напрямую или транзитивно), его содержимое вставляется в код несколько раз. Это нарушает ODR. Header guards предотвращают это.

    Проблема двойного включения

    Представь: main.cpp включает a.h и b.h, а оба они включают math.h. В итоге содержимое math.h вставляется дважды — ошибка переопределения.

    Два способа защиты

    C++ — header guards 
    // загрузка...
    #pragma once vs #ifndef
    • #pragma once — проще, одна строка, поддерживается всеми современными компиляторами (GCC, Clang, MSVC). Технически нестандартный, но де-факто стандарт.
    • #ifndef HEADER_H / #define HEADER_H / #endif — стандартный способ, работает везде. Имя макроса должно быть уникальным — обычно PROJECTNAME_FILENAME_H.
    • В новых проектах предпочтительнее #pragma once — меньше кода, нет риска опечатки в имени макроса.
