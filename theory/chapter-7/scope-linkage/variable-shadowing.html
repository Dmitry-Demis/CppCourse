    Перекрытие имён

    Когда переменная во вложенном блоке имеет то же имя, что и переменная во внешнем блоке, внутренняя «скрывает» внешнюю. Это называется перекрытием имён (name hiding или shadowing).

    Перекрытие локальных переменных

    C++ — перекрытие во вложенном блоке 
    #include <iostream>

int main()
{
    int apples { 5 }; // внешний apples

    {
        std::cout << apples << '\n'; // 5 — внешний apples ещё виден

        int apples { 0 }; // новый apples перекрывает внешний

        apples = 10; // изменяем ВНУТРЕННИЙ apples
        std::cout << apples << '\n'; // 10
    } // внутренний apples уничтожается

    std::cout << apples << '\n'; // 5 — внешний не изменился
    return 0;
}

    Вывод программы:

    Вывод
    5
10
5
    ⚠️ Важно

    Пока внутренний apples существует, к внешнему нет прямого доступа — он полностью скрыт.

    Перекрытие глобальных переменных

    Локальная переменная с тем же именем, что и глобальная, скрывает глобальную в своей области видимости. Но к глобальной всегда можно обратиться через :::

    C++ — доступ к глобальной через :: 
    #include <iostream>

int value { 5 }; // глобальная

int main()
{
    int value { 7 }; // скрывает глобальную
    ++value;         // изменяет локальную

    --(::value);     // изменяет глобальную через ::

    std::cout << "локальная: " << value << '\n';   // 8
    std::cout << "глобальная: " << ::value << '\n'; // 4
    return 0;
}

    Почему перекрытие опасно

    Перекрытие имён — частый источник трудноуловимых ошибок: вы думаете, что изменяете одну переменную, а на самом деле изменяете другую. Некоторые компиляторы выдают предупреждение при перекрытии.

    🔧 GCC/Clang

    Флаг -Wshadow включает предупреждения о перекрытии имён. Visual Studio включает их по умолчанию.

    Лучшая практика

    Избегайте перекрытия имён. Используйте префикс g_ для глобальных переменных — это автоматически устраняет большинство конфликтов.
