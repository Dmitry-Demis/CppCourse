    Категории значений: lvalue и rvalue

    Каждое выражение в C++ имеет два свойства: тип и категорию значения. Категория определяет, где выражение может использоваться — например, слева или справа от оператора присваивания.

    lvalue — «именованный объект»

    lvalue (left value) — выражение, которое ссылается на идентифицируемый объект, существующий дольше одного выражения. Проще говоря: если у выражения есть имя или адрес — это lvalue.

    C++ — примеры lvalue
    int x { 5 };       // x — lvalue (переменная с именем)
const double d {}; // d — lvalue (константная переменная)

int arr[3] {};
arr[0];            // arr[0] — lvalue (элемент массива)

int& ref { x };    // ref — lvalue (ссылка)
++x;               // ++x — lvalue (префиксный инкремент возвращает lvalue)

    rvalue — «временное значение»

    rvalue (right value) — выражение, которое вычисляется в значение, но не имеет постоянного адреса. Существует только в рамках выражения.

    C++ — примеры rvalue
    5;           // литерал — rvalue
3.14;        // литерал — rvalue
x + 1;       // результат операции — rvalue (временный объект)
x++;         // постфиксный инкремент — rvalue (возвращает копию)
getInt();    // возврат по значению — rvalue

    Почему это важно

    C++ — присваивание требует lvalue слева
    int x { 5 };
x = 10;  // OK: x — lvalue, можно присвоить
5 = x;   // ОШИБКА: 5 — rvalue, нельзя присвоить

// lvalue неявно конвертируется в rvalue когда нужно:
int y { x }; // x (lvalue) → rvalue (значение 5) → копируется в y
    💡 Быстрое правило

    Если выражение можно поставить слева от = — это lvalue. Если только справа — rvalue. Переменные — lvalue. Литералы и результаты вычислений — rvalue.
