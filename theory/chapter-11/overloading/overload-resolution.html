    Перегрузка функций

    Разрешение перегрузки и неоднозначные вызовы

    Когда аргументы вызова не совпадают точно ни с одной перегрузкой, компилятор проходит через последовательность шагов, пытаясь найти наилучшее совпадение.

    Шаги разрешения перегрузки

    C++ — шаги разрешения в действии 
    #include <iostream>

void foo(int x)    { std::cout << "foo(int)\n"; }
void foo(double x) { std::cout << "foo(double)\n"; }

int main()
{
    foo(5);     // Шаг 1: точное совпадение → foo(int)
    foo(5.0);   // Шаг 1: точное совпадение → foo(double)
    foo('a');   // Шаг 2: char повышается до int → foo(int)
    foo(true);  // Шаг 2: bool повышается до int → foo(int)
    foo(4.5f);  // Шаг 2: float повышается до double → foo(double)
    return 0;
}

    Неоднозначные вызовы

    Если на одном шаге компилятор находит несколько подходящих перегрузок — это неоднозначный вызов, ошибка компиляции:

    C++ — неоднозначный вызов 
    void foo(int x)    {}
void foo(double x) {}

int main()
{
    foo(5L); // long → int или long → double? Оба — числовые преобразования!
             // error: ambiguous call to overloaded function
    return 0;
}

    Как исправить неоднозначность

    C++ — три способа устранить неоднозначность 
    void foo(int x)    {}
void foo(double x) {}
void foo(long x)   {} // 1. Добавить точную перегрузку

int main()
{
    foo(5L);                        // теперь точное совпадение → foo(long)
    foo(static_cast<int>(5L));     // 2. Явное приведение типа
    foo(5);                         // 3. Использовать литерал нужного типа
    return 0;
}
