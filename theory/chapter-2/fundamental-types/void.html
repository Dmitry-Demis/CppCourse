Глава II — §2.1.1

Тип void и nullptr

В этом разделе мы рассмотрим два особых типа C++: void — пустой тип, не имеющий ни одного значения, и std::nullptr_t — тип безопасного нулевого указателя. Несмотря на кажущуюся простоту, они играют ключевую роль в системе типов языка.

1. Тип void — пустой тип

[basic.fundamental] §6.7.1 — ISO/IEC 14882 The void type has an empty set of values. It is an incomplete type that cannot be completed. Values of type void are not used.

void — это неполный тип (incomplete type): его нельзя завершить, нельзя создать объект этого типа, нельзя объявить ссылку на void. Применение void ограничено тремя чёткими контекстами:

Возврат функции void f()
Универсальный указатель void* ptr
Список параметров (C-совместимость) f(void)
⚠ Что запрещено с void

void x; — ошибка компиляции. void arr[10]; — ошибка. void& ref = …; — ошибка. Объект типа void не может существовать.

2. Функции, возвращающие void

Функции с типом возврата void — аналоги процедур в других языках. Они выполняют действия, но не возвращают значение. Оператор return; необязателен, но допустим.

C++ 
#include <iostream>
#include <string>

// Функция void — не возвращает значение
void greet(const std::string& name) {
    std::cout << "Привет, " << name << "!\n";
    return;  // необязательно
}

// Нельзя использовать результат void-функции
int main() {
    greet("Алексей");          // ✅ OK

    // int x = greet("Иван"); // ❌ void нельзя присвоить
    // std::cout << greet("Иван"); // ❌ тоже нельзя

    return 0;
}
📌 void в шаблонном программировании

Хотя void-объекты невозможны, тип void активно используется в шаблонном программировании: std::is_void<T>, специализации шаблонов, SFINAE-техники — всё это опирается на существование void как полноправного типа в системе типов.

3. Указатели на void — универсальный указатель

void* — обобщённый указатель: он может хранить адрес объекта любого типа. Именно на void* основаны функции C-библиотеки malloc, memcpy, qsort и другие.

⚠ void* нельзя разыменовать без явного приведения типа

Для разыменования указателя нужно знать тип данных. Поскольку void* этой информации не несёт, его нужно привести (static_cast или C-стиль) к конкретному типу прежде, чем обращаться к данным.

4. nullptr и std::nullptr_t

До C++11 нулевой указатель записывали как 0 или макрос NULL (обычно — (void*)0 или просто 0). Это порождало ошибки при перегрузке функций: компилятор выбирал f(int) вместо f(int*).

[lex.nullptr] §5.13.7 / [support.types.nullptr] §17.2.4 The pointer literal is the keyword nullptr. It is a prvalue of type std::nullptr_t.
typedef decltype(nullptr) nullptr_t;

nullptr vs NULL vs 0

C++ 
#include <iostream>

void func(int x)   { std::cout << "func(int):  " << x   << '\n'; }
void func(int* p)  { std::cout << "func(int*): " << p   << '\n'; }

int main() {
    func(0);       // ➡ func(int)  — 0 трактуется как int
    func(NULL);    // ➡ func(int)  — NULL == 0 на большинстве платформ
    func(nullptr); // ➡ func(int*) — nullptr — это указатель, не int

    return 0;
}

Свойства nullptr

Тип std::nullptr_t
Размер == sizeof(void*)
→ bool false
→ int ❌ ошибка
C++ 
#include <iostream>
#include <cstddef>   // std::nullptr_t

int main() {
    std::nullptr_t nptr{nullptr};

    // nullptr → любой тип указателя
    int*    iptr{nullptr};
    double* dptr{nullptr};
    void*   vptr{nullptr};

    // nullptr → bool (false)
    if (!nullptr) std::cout << "nullptr == false\n";

    // nullptr сравнивается с указателями
    if (iptr == nullptr) std::cout << "iptr — нулевой указатель\n";

    // Размер nullptr_t совпадает с размером указателя
    std::cout << "sizeof(nullptr_t) = " << sizeof(std::nullptr_t) << '\n';
    std::cout << "sizeof(void*)     = " << sizeof(void*)           << '\n';

    // int x = nullptr; // ❌ ошибка компиляции

    return 0;
}

5. Резюме

✅ Ключевые выводы

• void — неполный тип без значений; объекты void невозможны, но void* и void f() — ключевые конструкции C++.
• void* — обобщённый указатель; разыменование требует явного приведения типа.
• nullptr (C++11) заменяет NULL и 0; его тип — std::nullptr_t, который не приводится к int.
• sizeof(std::nullptr_t) == sizeof(void*).
