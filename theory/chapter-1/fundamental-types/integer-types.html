Глава I — §1.1.2

Целочисленные типы

Целочисленные типы — хребет большинства программ на C++: счётчики, индексы, идентификаторы, битовые маски. Стандарт гарантирует лишь минимальные размеры, поэтому важно знать, какой тип выбрать и чего ожидать на конкретной платформе.

[basic.fundamental] §6.7.1 — ISO/IEC 14882:2020 There are five standard signed integer types: signed char, short int, int, long int, and long long int. For each of the standard signed integer types, there exists a corresponding (but distinct) unsigned integer type.

1. Обзор семейства целочисленных типов

Тип Синонимы Мин. размер x86-64 (типично) Диапазон (signed)
signed char — 8 бит 1 байт −128 … 127
short int short ≥16 бит 2 байта −32 768 … 32 767
int signed ≥16 бит 4 байта −2 147 483 648 … 2 147 483 647
long int long ≥32 бита 8 б (Linux) / 4 б (Windows) платформенно-зависим
long long int C++11 long long ≥64 бита 8 байт −9.22×10¹⁸ … 9.22×10¹⁸

Для каждого знакового типа существует беззнаковый аналог (unsigned short, unsigned int и т. д.), сдвигающий диапазон в положительную область и удваивающий максимум.

2. Каждый тип — подробно

2.1 short

short — наименьший «полноценный» целый тип (не считая signed char). Используется там, где важна экономия памяти: большие массивы, сетевые пакеты, аудио-сэмплы.

short / signed short int
Размер (гарантия) ≥ 16 бит
Размер (x86-64) 2 байта
Диапазон −32 768 … 32 767
Суффикс литерала нет (неявное сужение)
Аудио-сэмплы (PCM 16-bit), компактные массивы, сетевые поля фиксированного размера.
unsigned short
Размер (x86-64) 2 байта
Диапазон 0 … 65 535
Порты TCP/UDP (0–65535), пиксельные координаты, USB HID-отчёты.
C++ — short 
#include <iostream>
#include <limits>

int main() {
    short          s1{32'767};           // C++14: digit separator
    unsigned short s2{65'535u};

    std::cout << "short max = "   << std::numeric_limits<short>::max()          << '\n';
    std::cout << "ushort max = "  << std::numeric_limits<unsigned short>::max() << '\n';
    std::cout << "sizeof(short) = " << sizeof(short) << " bytes\n";

    // Brace-init защищает от сужающего преобразования
    // short bad{100000};  // ❌ ошибка компиляции — 100000 не влезает в short

    return 0;
}

2.2 int

int — «естественный» целочисленный тип платформы. Стандарт гарантирует минимум 16 бит, но на всех современных платформах он 32-битный. Это тип по умолчанию для целочисленных литералов, счётчиков и арифметики.

int / signed int
Размер (гарантия) ≥ 16 бит
Размер (x86-64) 4 байта / 32 бита
INT_MIN −2 147 483 648
INT_MAX +2 147 483 647
Счётчики циклов, индексы, флаги, возвращаемые значения функций.
unsigned int
Размер (x86-64) 4 байта
Диапазон 0 … 4 294 967 295
Суффикс 42u или 42U
Битовые маски, хеш-значения, счётчики без отрицательных значений.
C++11/17/20/23 — int 
#include <iostream>
#include <limits>
#include <format>   // C++20
#include <print>    // C++23

int main() {
    int x{42};
    int y{-100};

    // C++11: uniform initialization защищает от сужения
    // int bad{3.14}; // ❌ ошибка — narrowing conversion

    // C++14: digit separators для читаемости
    int population{8'100'000'000};  // ❌ не влезет в int!
    // int population{2'000'000'000}; // ✅ влезает

    int big{2'000'000'000};
    std::cout << "big = " << big << '\n';           // C++03-compatible

    // C++17: if с инициализатором
    if (int result{big + 1}; result < 0)
        std::cout << "Переполнение!\n";

    // C++20: std::format — форматированный вывод
    std::string s = std::format("INT_MAX = {:d}", std::numeric_limits<int>::max());
    std::cout << s << '\n';

    // C++23: std::print — прямо в stdout без cout
    std::print("sizeof(int) = {} bytes\n", sizeof(int));

    return 0;
}

2.3 long

long — источник многих сюрпризов: его размер платформенно-зависим. На 64-битном Linux он 8 байт, на Windows — 4 байта. Это делает long нежелательным для переносимого кода; предпочтительнее int или long long.

long / long int
Размер (гарантия) ≥ 32 бита
Linux x86-64 8 байт (64 бит)
Windows x64 4 байта (32 бит)
Суффикс 42L или 42l
POSIX API (off_t, pid_t). Избегайте в переносимом коде — используйте int64_t.
unsigned long
Linux x86-64 0 … 18 446 744 073 709 551 615
Windows x64 0 … 4 294 967 295
Суффикс 42UL
В старом POSIX-коде — size_t и подобные системные типы.
⚠ Ловушка с long на разных платформах

Код, который работает на Linux (где sizeof(long) == 8), может вести себя иначе на Windows (где sizeof(long) == 4). Никогда не предполагайте, что long — это 64 бита. Для гарантированного 64-битного типа используйте long long или int64_t.

C++ — long и платформенные различия 
#include <iostream>
#include <limits>

int main() {
    long a{1'000'000'000L};
    unsigned long b{0xFFFF'FFFF'UL};

    std::cout << "sizeof(long) = " << sizeof(long) << " bytes\n";
    std::cout << "long max = " << std::numeric_limits<long>::max() << '\n';

    // C++17: static_assert с сообщением
    // static_assert(sizeof(long) == 8, "Ожидается 64-бит платформа (Linux)");

    return 0;
}

2.4 long long C++11

long long — единственный целочисленный тип, гарантирующий не менее 64 бит на любой поддерживаемой платформе. Его диапазон: $-2^{63}$ до $2^{63}-1$ ≈ ±9.2 × 10¹⁸. Используйте его всегда, когда значение может превысить ~2 млрд.

long long C++11
Размер (гарантия) ≥ 64 бита
LLONG_MIN −9 223 372 036 854 775 808
LLONG_MAX +9 223 372 036 854 775 807
Суффикс 42LL или 42ll
Unix timestamps, ID записей в БД, файловые смещения (>4 ГБ), астрономические расчёты.
unsigned long long
Диапазон 0 … 18.4 × 10¹⁸
ULLONG_MAX 18 446 744 073 709 551 615
Суффикс 42ULL
Криптография, хеш-значения, 64-битные битовые маски.
C++11/14/20/23 — long long 
#include <iostream>
#include <limits>
#include <format>   // C++20
#include <print>    // C++23

int main() {
    // ── C++11: brace-init, long long guaranteed 64-bit ──
    long long unixMs{1'710'000'000'000LL};   // C++14: digit separators

    // ── C++11: static_assert для compile-time проверки ──
    static_assert(sizeof(long long) >= 8, "long long must be at least 8 bytes");

    // ── ЛОВУШКА: переполнение до расширения типа ──
    int a{100'000};
    int b{100'000};
    long long wrong{a * b};          // ❌ a*b — это int*int = UB (overflow)
    long long right{(long long)a * b}; // ✅ один операнд long long — OK

    std::cout << "wrong = " << wrong << '\n';
    std::cout << "right = " << right << '\n';

    // ── C++20: std::format для форматированного вывода ──
    std::string msg = std::format(
        "LLONG_MAX = {:L}",          // 'L' — локальный разделитель тысяч
        std::numeric_limits<long long>::max()
    );
    std::cout << msg << '\n';

    // ── C++23: std::print (без явного cout) ──
    std::print("Unix timestamp (ms): {:d}\n", unixMs);

    return 0;
}

3. Модификаторы signed и unsigned

Добавление ключевого слова unsigned к любому целочисленному типу удваивает максимальное значение, но исключает отрицательные числа.

signed int (4 байта) −2 147 483 648 … +2 147 483 647
unsigned int (4 байта) 0 … 4 294 967 295
⚠ Ловушка: смешение signed и unsigned

При арифметике со смешанными типами компилятор неявно преобразует знаковый операнд к беззнаковому. Число -1 превращается в огромное положительное (UINT_MAX = 4 294 967 295).

C++ — unsigned trap 
#include <iostream>
#include <limits>
#include <format>   // C++20

int main() {
    unsigned int umax{std::numeric_limits<unsigned int>::max()};

    // ❌ Ловушка: -1 приводится к unsigned int = UINT_MAX = 4294967295
    if (-1 > umax)
        std::cout << "Кажется, -1 больше UINT_MAX?!\n"; // выполнится!

    // ✅ Правильно: привести к общему знаковому типу
    if (static_cast<long long>(-1) < static_cast<long long>(umax))
        std::cout << "OK: -1 < UINT_MAX\n";

    // C++20: format с числами
    std::print("umax = {}\n", umax);

    return 0;
}

4. Переполнение целочисленных типов

Целые числа хранятся в фиксированном числе битов. Если результат выходит за пределы диапазона, происходит переполнение — и поведение зависит от знаковости типа.

⚠ Переполнение signed — неопределённое поведение (UB)

По стандарту переполнение знакового целого — это UB. Компилятор вправе генерировать любой код. Оптимизаторы активно используют этот факт: они могут удалять проверки, предполагая, что UB никогда не происходит.

Для беззнаковых типов переполнение строго определено — результат берётся по модулю $2^n$ (wrap-around), что иногда намеренно используют в криптографии.

C++11/20 — overflow 
#include <iostream>
#include <limits>
#include <print>    // C++23

int main() {
    // ── Unsigned overflow: defined wrap-around ──
    unsigned int u{std::numeric_limits<unsigned int>::max()};
    ++u;   // 4294967295 + 1 = 0 (по модулю 2^32)
    std::print("unsigned wrap: {}\n", u);   // 0

    // ── Signed overflow: UB! ──
    int s{std::numeric_limits<int>::max()};
    // ++s;  // ❌ UB — компилятор вправе делать что угодно

    // ✅ Безопасная проверка перед операцией (C++11)
    if (s < std::numeric_limits<int>::max()) {
        ++s;
    } else {
        std::print("Переполнение предотвращено!\n");
    }

    // ✅ C++20:  - std::has_single_bit, std::popcount и др.
    // Для detached overflow detection см. __builtin_add_overflow (GCC/Clang)
    int x{2'000'000'000};
    int y{2'000'000'000};
    int result{};
    if (__builtin_add_overflow(x, y, &result))
        std::print("Overflow detected: {} + {} overflows int\n", x, y);

    return 0;
}

5. Типы с фиксированным размером — <cstdint>

Когда точный размер критичен (бинарные форматы, сетевые протоколы, аппаратные регистры), используйте типы из <cstdint>. Они гарантируют точный размер или возвращают ошибку компиляции, если платформа его не поддерживает.

Тип Размер Знаковость Диапазон / max Пример применения
int8_t 8 бит signed −128 … 127 Аппаратные регистры, компактные флаги
uint8_t 8 бит unsigned 0 … 255 Байт данных, пиксель, байт буфера
int16_t 16 бит signed −32 768 … 32 767 Аудио-сэмпл PCM
uint16_t 16 бит unsigned 0 … 65 535 Порт TCP/UDP, Unicode codepoint (BMP)
int32_t 32 бита signed −2.1×10⁹ … 2.1×10⁹ IPv4-адрес, цвет ARGB
uint32_t 32 бита unsigned 0 … 4.3×10⁹ CRC32, хеш-значения
int64_t 64 бита signed ±9.2×10¹⁸ Unix timestamp, ID записей в БД
uint64_t 64 бита unsigned 0 … 18.4×10¹⁸ 64-битная маска, большой счётчик
intptr_t ≥ ptr size signed платформенный Хранение указателя как числа
std::size_t ≥ ptr size unsigned 0 … платформенный Размеры объектов, индексы контейнеров
C++11/17/20 — <cstdint> 
#include <iostream>
#include <cstdint>
#include <format>   // C++20
#include <bit>      // C++20: std::popcount, std::byteswap

// Бинарный заголовок файла: каждое поле — точный размер
struct BitmapFileHeader {
    uint16_t signature;    // "BM" = 0x424D
    uint32_t fileSize;
    uint32_t reserved;
    uint32_t dataOffset;
};

int main() {
    // C++11: static_assert с brace-init
    static_assert(sizeof(BitmapFileHeader) == 14, "Неверный размер структуры!");

    // C++14: digit separators
    uint64_t timestamp{1'710'000'000'000ULL};  // миллисекунды с Unix epoch

    // C++17: structured bindings (для примера с парой)
    auto [lo, hi] = std::pair{uint32_t{0xDEAD'BEEF}, uint32_t{0xCAFE'BABE}};

    // C++20: std::popcount — количество единичных бит
    uint32_t mask{0b1010'1010'1010'1010u};
    int bits{static_cast<int>(std::popcount(mask))};

    // C++20: std::format
    std::print("timestamp = {}\n", timestamp);
    std::print("popcount(mask) = {}\n", bits);
    std::print("lo = {:#010x}, hi = {:#010x}\n", lo, hi);

    // Правильный тип для индексов и размеров
    std::size_t arraySize{1024};
    for (std::size_t i{0}; i < arraySize; ++i) { /* ... */ }

    return 0;
}
💡 Рекомендации по выбору типа
  • Используйте int по умолчанию — это «естественный» тип платформы.
  • Используйте long long если значения могут превысить ~2 млрд.
  • Используйте int32_t / uint8_t и др. для бинарных протоколов и форматов.
  • Используйте std::size_t для размеров и индексов контейнеров.
  • Избегайте long в переносимом коде — его размер непредсказуем на разных платформах.
  • Избегайте unsigned там, где возможны смешанные вычисления со знаковыми типами.

6. std::numeric_limits — интроспекция типов

Шаблон std::numeric_limits<T> из <limits> даёт compile-time доступ к свойствам любого числового типа — лучший способ узнать диапазон переносимо.

C++11/20/23 — numeric_limits 
#include <iostream>
#include <limits>
#include <cstdint>
#include <print>    // C++23

template<typename T>
void printTypeInfo(const char* name) {
    using nl = std::numeric_limits<T>;
    std::print("{:20s}: size={} bytes, min={:20}, max={}\n",
        name,
        sizeof(T),
        nl::min(),
        nl::max()
    );
}

int main() {
    printTypeInfo<short>("short");
    printTypeInfo<int>("int");
    printTypeInfo<long>("long");
    printTypeInfo<long long>("long long");
    printTypeInfo<unsigned int>("unsigned int");
    printTypeInfo<unsigned long long>("unsigned long long");
    printTypeInfo<int32_t>("int32_t");
    printTypeInfo<int64_t>("int64_t");

    return 0;
}
