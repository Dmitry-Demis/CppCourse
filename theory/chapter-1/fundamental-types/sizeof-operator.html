Глава I — §1.1.9

Оператор sizeof

Размер типа данных в памяти — одна из ключевых характеристик, влияющих на производительность и корректность программы. C++ предоставляет встроенный оператор sizeof для определения размера типов и объектов на этапе компиляции.

1. Что такое sizeof

sizeof — унарный оператор, возвращающий размер типа данных или объекта в байтах. Результат имеет тип std::size_t — беззнаковый целочисленный тип, определённый в <cstddef>.

Стандарт ISO/IEC 14882: [expr.sizeof] §7.6.2.5

Оператор sizeof может применяться к:

  • Типам данных: sizeof(int), sizeof(double)
  • Переменным: sizeof(x), sizeof(array)
  • Выражениям: sizeof(x + y) (выражение не вычисляется!)

2. Синтаксис

C++ 
sizeof(тип)
sizeof выражение
sizeof(выражение)

Примеры:

C++ 
#include <iostream>

int main() {
    // Размер типов
    std::cout << "sizeof(bool):       " << sizeof(bool)       << " байт\n";
    std::cout << "sizeof(char):       " << sizeof(char)       << " байт\n";
    std::cout << "sizeof(short):      " << sizeof(short)      << " байт\n";
    std::cout << "sizeof(int):        " << sizeof(int)        << " байт\n";
    std::cout << "sizeof(long):       " << sizeof(long)       << " байт\n";
    std::cout << "sizeof(long long):  " << sizeof(long long)  << " байт\n";
    std::cout << "sizeof(float):      " << sizeof(float)      << " байт\n";
    std::cout << "sizeof(double):     " << sizeof(double)     << " байт\n";
    std::cout << "sizeof(long double):" << sizeof(long double)<< " байт\n";

    // Размер переменных
    int x = 42;
    double pi = 3.14159;
    
    std::cout << "\nsizeof(x):  " << sizeof(x)  << " байт\n";
    std::cout << "sizeof(pi): " << sizeof(pi) << " байт\n";

    return 0;
}
Типичный вывод на 64-битной системе
sizeof(bool):       1 байт
sizeof(char):       1 байт
sizeof(short):      2 байт
sizeof(int):        4 байт
sizeof(long):       8 байт
sizeof(long long):  8 байт
sizeof(float):      4 байт
sizeof(double):     8 байт
sizeof(long double):16 байт

3. Ключевые особенности

3.1. Размер зависит от платформы

Стандарт C++ не фиксирует точные размеры типов (кроме char). Размеры зависят от архитектуры процессора, операционной системы и компилятора:

  • int может быть 2 или 4 байта (обычно 4)
  • long может быть 4 или 8 байт (4 на Windows x64, 8 на Linux x64)
  • long double может быть 8, 10, 12 или 16 байт
⚠ Портируемость кода

Никогда не полагайтесь на конкретные размеры типов в переносимом коде! Используйте sizeof для динамического определения размеров или типы фиксированной ширины из <cstdint>: int32_t, uint64_t и т.д.

3.2. sizeof(char) всегда равен 1

По определению стандарта C++, sizeof(char), sizeof(signed char) и sizeof(unsigned char) всегда равны 1. Это не означает, что байт содержит 8 бит — количество бит в байте определяется макросом CHAR_BIT из <climits> (обычно 8, но может быть больше на экзотических платформах).

C++ 
#include <iostream>
#include <climits>  // для CHAR_BIT

int main() {
    std::cout << "sizeof(char) = " << sizeof(char) << " байт\n";
    std::cout << "CHAR_BIT = " << CHAR_BIT << " бит\n";
    std::cout << "Размер char в битах: " << sizeof(char) * CHAR_BIT << " бит\n";
    return 0;
}

3.3. sizeof не вычисляет выражение

Оператор sizeof определяет размер типа на этапе компиляции. Если применить его к выражению, само выражение не вычисляется:

C++ 
#include <iostream>

int main() {
    int x = 5;
    
    std::cout << "sizeof(x++) = " << sizeof(x++) << '\n';
    std::cout << "x = " << x << '\n';  // x всё ещё равен 5!
    
    // sizeof определяет тип выражения (int), но не выполняет x++
    
    return 0;
}
Почему sizeof не вычисляет выражение?

Компилятор определяет тип выражения x++ (это int), затем возвращает sizeof(int). Само выражение не нужно вычислять — достаточно знать его тип. Это позволяет использовать sizeof с выражениями, которые невозможно вычислить на этапе компиляции.

3.4. sizeof возвращает size_t

Результат sizeof имеет тип std::size_t — беззнаковый целочисленный тип, способный представить размер любого объекта в памяти. На 32-битных системах это обычно unsigned int (4 байта), на 64-битных — unsigned long long (8 байт).

C++ 
#include <iostream>
#include <cstddef>  // для std::size_t

int main() {
    std::size_t size = sizeof(int);
    std::cout << "Размер int: " << size << " байт\n";
    std::cout << "sizeof(size_t): " << sizeof(std::size_t) << " байт\n";
    return 0;
}

4. Практическое применение

4.1. Определение количества элементов в массиве

Классический приём для вычисления длины массива:

C++ 
#include <iostream>

int main() {
    int array[] = { 10, 20, 30, 40, 50 };
    
    // Размер всего массива / размер одного элемента
    int length = sizeof(array) / sizeof(array[0]);
    
    std::cout << "Длина массива: " << length << '\n';
    // Вывод: Длина массива: 5
    
    return 0;
}
⚠ Массивы и указатели

Этот приём работает только для массивов, объявленных локально. Если массив передан в функцию как параметр, он «распадается» в указатель, и sizeof вернёт размер указателя (обычно 8 байт на 64-битной системе), а не размер массива!

4.2. Проверка размеров на конкретной системе

C++ 
#include <iostream>

int main() {
    std::cout << "Размеры типов на этой системе:\n";
    std::cout << "int:     " << sizeof(int)     << " байт\n";
    std::cout << "pointer: " << sizeof(void*)   << " байт\n";
    std::cout << "size_t:  " << sizeof(size_t)  << " байт\n";
    
    return 0;
}

4.3. Работа с динамической памятью

sizeof часто используется при выделении памяти:

C++ 
#include <iostream>
#include <cstdlib>  // для malloc/free

int main() {
    // Выделение памяти для 10 элементов типа int
    int* array = (int*)malloc(10 * sizeof(int));
    
    if (array != nullptr) {
        for (int i = 0; i < 10; ++i) {
            array[i] = i * i;
        }
        
        for (int i = 0; i < 10; ++i) {
            std::cout << array[i] << ' ';
        }
        std::cout << '\n';
        
        free(array);
    }
    
    return 0;
}

5. Важные замечания

Ключевые моменты
  • sizeof — оператор времени компиляции, не функция
  • Результат имеет тип std::size_t
  • sizeof(char) всегда равен 1 по определению
  • Размеры типов зависят от платформы
  • Выражение в sizeof не вычисляется
  • Для массивов возвращает общий размер всех элементов
  • Для указателей возвращает размер самого указателя, а не объекта
ISO/IEC 14882 — [expr.sizeof] §7.6.2.5 «The sizeof operator yields the number of bytes occupied by a non-potentially-overlapping object representation of its operand. The operand is either an expression, which is an unevaluated operand, or a parenthesized type-id.»

Заключение

Оператор sizeof — фундаментальный инструмент для работы с памятью в C++. Он позволяет писать переносимый код, не зависящий от конкретных размеров типов на разных платформах. Понимание работы sizeof критически важно для эффективного управления памятью и оптимизации производительности.
